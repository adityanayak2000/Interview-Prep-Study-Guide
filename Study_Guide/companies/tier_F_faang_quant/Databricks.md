# Databricks
**Role(s):** Software Engineer (India)  
**Tier:** F  
**Status:** done-v3

## Process Overview
- Recruiter screen (~15 min)
- OA for some pipelines (not universal — some skip to phone screen)
- Technical phone screen: DSA-focused
- Virtual onsite: ~4 rounds — 2 algorithm/coding, 1 concurrency/multithreading round, 1 behavioral; some add systems/infra discussion
- Hiring manager / team-fit round

## Online Assessment (OA)
- Platform unknown. Format when present: coding task producing compiled/running code, sometimes OOP design patterns. Negative marking unknown. Difficulty leans hard (LeetCode-hard more common than medium).

## Interview Rounds
- Coding (x2): DSA — data structure design (SluggishArray w/ map+getIndexOf), IP-to-CIDR, variable-size tic-tac-toe
- Concurrency/multithreading round: distinct standalone pillar — thread-safe structures, locks, producer-consumer
- Systems knowledge: SSL/TCP-IP, S3/AWS/GCP, Kubernetes/Docker/storage for some infra-adjacent roles
- Behavioral round with hiring manager/team

## Topic Weights
| DSA | OS | C/C++ | DBMS | CN | LLD | HLD | Concurrency | Aptitude |
|---|---|---|---|---|---|---|---|---|
| High | Med | Med | Low | Med | Med | Low | High | None |

## Recurring Patterns & Tips
- Practice LeetCode-hard, not just mediums
- Dedicate specific prep time to concurrency/multithreading (thread-safe structures, producer-consumer)
- Review distributed systems/cloud basics if role touches Spark/data platform
- Expect working, compilable code in OA/coding rounds, not pseudocode
