# Quantbox Research
**Role(s):** Quant / SWE (India)  
**Tier:** F  
**Status:** insufficient

## Process Overview
- Unconfirmed order, pieced together: resume/CV shortlist (low hiring volume) → online screening test (proctored, HackerRank-style per one report) → 2-3 back-to-back technical/coding interviews → possible probability/quant-math round → final round/offer.

## Online Assessment (OA)
- Platform: HackerRank reported in one account, unconfirmed as standard. Duration unknown. Format per one report: 3 sections (~21 objective/short Qs each) on OOP/Probability/Networks + 1 section with 2 medium-hard coding problems. Negative marking unknown. Treat as anecdotal, not verified official format.

## Interview Rounds
- Coding: LeetCode medium/hard (graph/array problems), sometimes live-coded without IDE/compiler
- Practical/systems: order-book-like logic implementation (light HLD/domain-simulation flavor)
- Quant/math: probability puzzles, Markov chains, combinatorial game theory (Nim variants)
- CS fundamentals: OOP, networking basics (from OA)

## Recurring Patterns & Tips
- Practice LeetCode medium/hard (graphs, arrays, "next greater element"-style) — interviewers reportedly reuse known problems
- Brush up probability puzzles and game theory (Markov chains, Nim)
- Be ready to code without IDE/autocomplete in some rounds
- Given thin public data, supplement with general HFT/quant-dev prep rather than assuming a fixed format
