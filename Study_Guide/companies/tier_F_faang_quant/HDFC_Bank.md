# HDFC Bank
**Role(s):** SWE / technology (India — post-merger bank tech)  
**Tier:** F  
**Status:** done-v3

_Caveat: Distinct from pre-merger HDFC Ltd. Process lean relies partly on aggregator write-ups over dense GFG-native SDE threads._

## Process Overview
- Online aptitude test
- Technical/coding interview (sometimes MS Teams w/ HackerRank-style pad)
- System-design/deep-dive round with senior engineer (some tracks)
- HR round (final)
- Freshers: typically slimmer (1 technical + 1 HR); experienced hires: 2 technical + 1 HR

## Online Assessment (OA)
- Platform unknown (one report: 60-min live coding + technical interview on MS Teams via HackerRank-style pad — arguably a live round, not async OA). Negative marking unknown. Difficulty moderate; explicitly "solid fundamentals, not CP-level."

## Interview Rounds
- Coding: simple algorithmic tasks (e.g. first non-repeating character), follow-up on complexity/edge cases
- SQL: queries against sample banking schemas (accounts, transactions)
- Tech stack: Java, JavaScript, Spring Boot, REST APIs, React
- System/design: light discussion in deep-dive round
- Behavioral: teamwork, integrity, customer focus (stated bank values), final HR

## Recurring Patterns & Tips
- Focus on SQL fluency against realistic banking schemas — recurs as a concrete ask
- Solidify Java + Spring Boot + REST API basics; framework literacy matters more than algorithmic depth
- Practice easy-medium LeetCode, discuss complexity/edge cases conversationally
- Emphasize teamwork/integrity/customer-focus in behavioral answers
