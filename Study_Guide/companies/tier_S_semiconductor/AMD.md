# AMD
**Role(s):** Software Engineer, Engineer Trainee, Intern (CS / dual-degree), Senior Software (experienced / Xilinx-side)  
**Tier:** S  
**Status:** done-v3

## Process Overview
Less standardized than Qualcomm/TI. Campus software paths may shortlist on resume then interview on Teams; trainee/core tracks use long written tests; experienced software loops often run 3–4 technical rounds and sometimes a take-home after an early screen. Expect earlier system-design / cohesion–coupling talk than at pure firmware MCQ companies.

## Online Assessment (OA)
- **Platform:** Varies by drive — may be none (resume → interview), offline/online written for trainee/hardware, or coding OA / take-home for software.
- **Duration:** Trainee mains historically ~3 hours; software take-homes and timed coding sets vary widely.
- **Sections & format (software-leaning):** CS fundamentals, DSA, OOP (C++/Java/Python); some candidates report medium–hard coding or DP-style take-homes after a first technical. Trainee papers: aptitude, C, digital/logic, microprocessors (Verilog on some).
- **Negative marking:** Drive-dependent; not a universal HirePro-style constant.
- **Topics tested:** Lists/stacks/queues, OS/memory, OOP, light system design; hardware electives for core roles.
- **Difficulty:** Medium overall; process variance is higher than peer semiconductor OAs.

## Interview Rounds
- **Resume / project walkthrough:** Central on nearly every report (intern and FTE).
- **DSA coding:** Linked-list insert/middle, trees, matrices, string rotations, intersection of lists — shared editor or live coding.
- **Systems:** Threads vs processes, sync, virtual memory, caches, boot/loader questions on CS intern panels.
- **Design / SE:** Cohesion & coupling, basic system design, occasional DB design — appears earlier than at TI/Qualcomm for software roles.
- **HR / managerial:** Relocation, strengths/weaknesses, behavioral; Glassdoor aggregates rate SE difficulty around mid-pack.

## Recurring Patterns & Tips
- Do not assume a single HirePro template — ask recruiters which format your drive uses.
- For software teams, prep **system design lite** earlier than for Qualcomm/TI.
- C/C++ memory and pointer correctness still matter on CS intern panels.
- Use the local OA Drive folder for practice artifacts when available.
