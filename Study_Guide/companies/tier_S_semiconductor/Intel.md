## Process Overview
Often resume/CGPA shortlist then 2–3 technical rounds + HR. Some older drives used eLitmus. Strong preference for C/C++ or Python over Java for many Intel software teams.

## Online Assessment (OA)
- **Platform:** Varies; many campus reports skip a heavy OA and go resume → interviews. When present: technical CBT / eLitmus historically.
- **Topics:** C/C++ pointers, OOP, OS, basic DSA.
- **Difficulty:** Easy–Medium coding when asked.

## Interview Rounds
- Resume/project deep-dive (expect grilling on everything listed).
- Coding: e.g. 2-sum, linked-list palindrome, level-order, reverse string without library.
- C++: pointers, memory spaces, malloc/new, virtual functions, static.
- OS: scheduling, paging, threads, race conditions.
- HR: behavioral, higher studies, location.
