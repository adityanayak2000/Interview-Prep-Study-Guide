# Arm
**Role(s):** Software / Embedded Software FTE; Intern; some HireVue-screened embedded paths  
**Tier:** S  
**Status:** done-v3

## Process Overview
Campus FTE drives typically open an OA to CS/CE/ECE branches, then run online technical interviews heavy on low-level C/C++. Arm’s IP/architecture focus shows up as Computer Architecture & Organization (CAO) in the OA and as memory hierarchy / pointer / bitmask questions in interviews. Some embedded tracks use HireVue video screens before or instead of a classic proctored OA.

## Online Assessment (OA)
- **Platform:** Campus OA portals (varies by college drive); coding expected in **C or C++**.
- **Duration:** Drive-dependent; results often same-day for campus.
- **Sections & format:** Fundamental CS MCQs; CAO / architecture questions; typically **two coding problems** in C/C++ (not Java-first).
- **Negative marking:** Not consistently reported.
- **Topics tested:** CS fundamentals, computer architecture/organization, C/C++ coding.
- **Difficulty:** Medium — architecture literacy and C fluency matter as much as algorithms.

## Interview Rounds
- **Technical (resume-led):** C/C++ memory management, pointers, OOP (inheritance, polymorphism, encapsulation).
- **Live coding:** String reverse via pointers; bitmask / binary manipulation tasks.
- **Systems / architecture:** Paging, virtual memory, MMU, caches, sync — depending on team; ARM-relevant CPU/pipeline talk on some panels.
- **HireVue (some embedded paths):** Timed spoken behavioral + light technical prompts.
- **HR / closing:** Fit, location, role interest (when separate).

## Recurring Patterns & Tips
- Prioritize **C pointers, bitmasks, and OS memory hierarchy** over Hard graph/DP sets.
- Brush caches, MMU, and virtual memory vocabulary — interviews reward systems language.
- Practice writing pointer-based string and bit utilities without relying on STL shortcuts.
- If HireVue appears, rehearse concise timed answers; content still skews low-level.
