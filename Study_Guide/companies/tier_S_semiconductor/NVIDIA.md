# NVIDIA
**Role(s):** System Software Engineer (SSE) Intern/FTE, SDE Intern (India campus)  
**Tier:** S (F-adjacent peer)  
**Status:** done-v3

## Process Overview
India on-campus: CGPA/marks shortlist → HackerRank OA → 1–2 technical interviews (sometimes single combined tech+HR). SSE tracks stress C/C++, OS, architecture; coding tracks love DP/graphs/tries.

## Online Assessment (OA)
- **Platform:** HackerRank (often proctored / lab).
- **Duration:** ~60–70 minutes common.
- **Sections:** ~2 coding (Medium–Hard; sometimes one must be C) + ~14–25 MCQs (aptitude, OS, C/C++ output, DSA, arch/probability).
- **Topics tested:** DP, graphs, tries, greedy, OS, pointers.
- **Difficulty:** Hard filter; few shortlisted per college.

## Interview Rounds
- Projects + live coding (trees, LL cycle, strings, bit manip).
- C++ internals / pointers / segfaults; OS (processes, sync).
- Sometimes HR-in-tech: Why NVIDIA / products / AI.
- Full-time loops (off-campus) can expand to more systems/parallelism rounds (cache coherence, C++17/20) — see library note in Interview Prep Resources.

## Recurring Patterns & Tips
- Always present brute force → optimize; don’t jump to known LC optimal silently.
- DP + Trie + Graph called out as NVIDIA favorites in campus reports.
- Study GPU/CUDA awareness for “why NVIDIA” without faking kernel expertise.
