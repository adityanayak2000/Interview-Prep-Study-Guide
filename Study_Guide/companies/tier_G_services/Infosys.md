## Process Overview
Infosys runs **parallel hiring tracks with different OA shapes** — do not collapse them into one “the Infosys OA.”

| Track | Typical shape |
|---|---|
| **System Engineer (mass campus)** | Aptitude / verbal / reasoning (± pseudocode or gamified sections) OA → combined or split Technical+HR |
| **SP / DSE** | Proctored coding OA on Infosys Wingspan (often framed via HackWithInfy) → technical(+HR) reviewing OA code + fundamentals → HR logistics |
| **Power Programmer** | OA → technical interviews → behavioral/HR — PP-specific round detail is thinly sourced |
| **InfyTQ** | Certification/screening exam that can feed some pipelines — not itself a full interview loop |

Campus SE drives can be ~1–2 days (orientation + OA, then interviews for shortlisted candidates).

_Caveat: medium-high confidence overall, but OA section timing/marking varies by year — report variants; don’t invent a single canonical timer._

## Online Assessment (OA)
### SE variants (genuinely different across years)
- **Variant A:** ~95 min — Quant, Logical, Verbal with sectional timing and no cross-section switching.
- **Variant B (2024 on-campus):** ~90 min / ~50 Q — Quant, Numerical, English, Essay, Pattern, Gamified assessment; **no negative marking**; no sectional limit reported.
- **Variant C (2025 secondary):** ~100 min, five sections (Math, Logical, Verbal, Pseudocode, Puzzles) with sectional cutoffs, no negative marking, no cross navigation.
- **Pseudocode MCQs** appear on the SE/general track (trace C/C++/DBMS/DS/SQL-style logic). SP/DSE uses live coding instead.

### SP / DSE
- **Platform:** Infosys platform / Wingspan (named in SP/DSE accounts).
- **Format:** 3 hours, 3 coding problems (easy/medium/hard — arrays/two-pointers, DP, graphs common). Proctored; plagiarism checks. Negative marking not found for this variant.
- Rough score mapping reported: ~2+ fully solved → SP eligibility; ~1–1.5 → DSE (indicative, not a published universal rule).

### InfyTQ (secondary)
- Qualifier MCQs with reported +1/−0.25; Final mix of MCQs + Python coding — treat as certification screening, not the SE OA.

## Interview Rounds
- **DSA:** Arrays/strings, DP (Knapsack/LCS-family), graphs (BFS/Dijkstra/cycle), hashing, binary search — heavier on SP/DSE.
- **CS fundamentals:** OOP, DBMS/SQL (live demos), OS, CN; language/framework follow-ups tied to resume.
- **Projects:** Core for SE and SP/DSE alike.
- **HLD/LLD:** Not found in verified first-person fresher accounts.
- **HR:** Fit, communication, relocation.

## Topic Weights

|---|---|---|---|---|---|---|---|---|
| High (SP/DSE) / Med (SE) | Med | Med | Med–High | Low–Med | Low / Unknown | Unknown | Low | High (SE) |

## Recurring Patterns & Tips
- Identify your track before grinding: SE aptitude/pseudocode vs Wingspan 3-problem coding.
- SP/DSE: be ready to defend OA code line-by-line in the interview.
- SE: practice sectional aptitude + pseudocode tracing under time.
- Projects matter on every fresher track.
