# CynLr
**Role(s):** Software Engineer (robotics / industrial computer-vision; Bengaluru)  
**Tier:** G  
**Status:** done-v3

## Process Overview
Glassdoor-heavy sample: intro/background call → coding/knowledge assessment → technical interview → founders’ round → offer. Campus-leaning accounts add MCQ shortlist → coding (~1.5 hr) → DSA technical ± multi-day take-home. Feedback cycles often described as slow.

_Caveat: medium confidence — mostly Glassdoor; limited independent GFG corroboration._

## Online Assessment (OA)
- **Platform:** Unknown.
- **Formats reported:** Long subjective/written mix (~3 hr in one account: physics intuition + AI/robotics opinions) and/or separate MCQ + coding for campus hires.
- **Negative marking:** Unknown.

## Interview Rounds
- DSA examples: string occurrence, queue implementation — medium emphasis.
- Stronger signal: computer organization, compilation, bits/bytes, first-principles reasoning over one-word algorithm trivia.
- Take-home project stage possible — budget calendar time.
