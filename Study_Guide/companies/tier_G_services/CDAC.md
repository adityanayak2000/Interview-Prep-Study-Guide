# C-DAC
**Role(s):** Project Engineer (PE); Project Associate (PA); Project Intern; Scientist B / Group A (higher tracks)  
**Tier:** G  
**Status:** done-v3

## Process Overview
Centre for Development of Advanced Computing runs **two distinct pipelines that must not be mixed**:

1. **C-CAT (CDAC-ACTS)** — academic admission test for PG Diploma courses (PG-DAC, PG-DESD, etc.). **Not a job-hiring OA.**
2. **PE/PA (and related) recruitment** — the actual employment track: apply/resume screen → **written MCQ** (role syllabus on the advertisement; negative marking often official) → 1–2 technical panel interviews → HR/PI → offer.

Intern hiring can be lighter (shortlist → one technical). Scientist B notifications may keep written/interview formats “to be updated” on the site — treat those cycles as advertisement-specific.

## Online Assessment (OA) / Written Exam
### Job track (PE/PA) — prefer official cdac.in notifications
- **Example PE (older Silchar-style PDF):** Objective MCQ, ~1 hour, aptitude/reasoning + programming (C/C++/Java) + DB/DS + web tech; **negative marking confirmed** (+2 / −1 in that advertisement).
- **Example PA System Admin track:** Shorter MCQ paper on OS/networking/sysadmin tooling; different negative-marking scale in that advertisement.
- **Other PE cycles:** Some secondary reports cite ~100 Q / longer windows with English/reasoning/aptitude/GA + one technical subject — **marking/format vary; always read the live advertisement.**
- **First-hand Bengaluru PE note:** ~100 MCQs / ~1–1.5 hr with negative marking; self-estimated mix of aptitude, core CS, and domain — single-source weight estimate only.
- **Platform:** Unknown.

### C-CAT (academic only — do not cite as PE OA)
- Sections A/B/(C for some courses), timed MCQs on aptitude + CS fundamentals (± digital/microprocessors for hardware tracks). Useful only if you are applying to ACTS courses — **not** as the PE hiring paper.

## Interview Rounds
- **CS fundamentals:** OOP pillars, DBMS/SQL (joins, second-highest salary, normalization), OS sync/deadlocks, networking/SSL.
- **DSA:** Basic–medium (list reverse, sliding window, tree/graph traversal, Dijkstra, 2-sum family) — not typically a Hard contest bar.
- **Language/stack:** Java collections/Spring or C/C++/REST depending on centre/role.
- **Projects:** Universally deep; scaling “what breaks at 10× users?” style follow-ups appear.
- **Domain (role/centre dependent):** HPC, embedded/RTOS, VLSI/FPGA, cybersecurity, AI/ML, quantum, Linux sysadmin — driven by the posting syllabus.
- **HR:** Communication and fit.

## Recurring Patterns & Tips
- Download the **current centre advertisement PDF** first — syllabus and negative marking are official there.
- Never treat C-CAT PG-Diploma prep as the PE hiring exam.
- Domain depth matters once shortlisted; match the lab/centre focus.
