# Schlumberger (SLB)
**Role(s):** Software Engineer / Intern / Full-stack (India tech centers Pune/Mumbai)  
**Tier:** G  
**Status:** done-v3

## Process Overview
Typical campus-leaning path: CGPA/CS shortlist → **online assessment (Java theory + code-output + coding)** → two technical rounds → TCM/manager + HR; some pipelines include Group Discussion.

## Online Assessment (OA)
- **Platform:** HackerRank in some accounts.
- **Format examples:** Java theory MCQs (~10/10 min) + Java code-output (~10/10 min) + coding (decimal→binary, Fibonacci, etc.); alternate “2 coding + 6 MCQ” reports.
- **Negative marking:** Unknown.
- **Difficulty:** Easy–moderate.

## Interview Rounds
- DSA (duplicates under constraints, sorting, string reverse).
- Full-stack lean: MongoDB/Firebase, basic Node.js in some reports.
- Resume/projects; GD communication where present.

## Topic Weights
| DSA | OS | C/C++ | DBMS | CN | LLD | HLD | Concurrency | Aptitude |
|---|---|---|---|---|---|---|---|---|
| Med | Low | Low (Java Med) | Med | Low | None | None | None | Med |

## Recurring Patterns & Tips
- Practice Java output-prediction MCQs — they are a distinct scoreable OA chunk.
- Full-stack JDs need DB + Node beyond core DSA.
