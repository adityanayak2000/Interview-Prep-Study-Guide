## Process Overview
India campus reports converge on a multi-stage loop: **speed aptitude → online coding OA → live coding technical → second technical (language + SQL + projects)** with HR conversations interleaved rather than a single fixed last stage.

_Caveat: moderate confidence — OA vendor name unknown; no sourced eDiscovery/domain system-design questions despite the product space._

## Online Assessment (OA)
- **Platform:** Unknown.
- **Duration:** Aptitude ~15–16 min; coding/technical OA ~60 min (separate stages).
- **Sections & format:** Aptitude ≈50 speed/logic/algebra questions (informal ~40/50 cutoff mentioned once). Coding OA: ~2 medium coding problems plus programming MCQs (counts vary by account).
- **Negative marking:** Unknown.
- **Topics tested:** Arrays, recursion, sorting/searching; tags also cite graphs, DP, BST, heaps, grids, combinatorics.
- **Difficulty:** Aptitude is a hard speed filter; coding OA rated medium.

## Interview Rounds
- **Live coding (~40–60 min):** Medium–hard problem on a shared editor; interviewers often change requirements mid-problem and probe complexity.
- **Second technical:** Easy–medium DSA, then C++ fundamentals, threads/OOP/synchronization, SQL queries, internship/project deep-dive. Java is often preferred, but Python/C++ appear too. One SE review mentioned computer architecture.
- **HR:** Why ZL, relocation, career plans, light AI-impact questions.
- **HLD / eDiscovery domain:** Not found in fresher reports — do not invent indexing/search-pipeline content.

## Recurring Patterns & Tips
- Speed aptitude is a real first gate — accuracy under time beats deep algebra study.
- Expect requirement changes during live coding; narrate approach and complexity.
- Prep SQL + OOP/threads alongside DSA.
- Java fluency helps even if you code interviews in another language.
