# Fiorano
**Role(s):** Software Engineer (campus + internship); integration/ESB-adjacent engineering roles in some cycles  
**Tier:** P  
**Status:** done-v3

## Process Overview
Fiorano Software (integration middleware / ESB / JMS) runs a campus-style loop that usually looks like **OA → 1–2 technical rounds → optional HR**. Round count is not fixed (reports range 3–5). Recent Glassdoor (2025) leans OA → Tech R1 (basic DSA + resume) → Tech R2 (harder DSA/puzzles). Some older accounts skip a formal HR stage.

_Caveat: moderate confidence — single-digit detailed reports (2018–2025); treat timings/platform as batch-dependent._

## Online Assessment (OA)
- **Platform:** Unknown (no source names a vendor).
- **Duration:** ~75–90 minutes depending on batch.
- **Sections & format:** Commonly ~60 MCQs in three blocks of ~20 — Quantitative aptitude, Data Structures, and C **or** Java (candidate choice). At least one account also saw OS/DBMS items in the written test.
- **Negative marking:** Unknown.
- **Topics tested:** Quant, DS theory, C/Java language concepts; occasional core CS MCQs.
- **Difficulty:** Described as a tough MCQ paper; shortlist ratios can be steep (one batch shortlisted only 9 after Round 1).

## Interview Rounds
- **Technical (DSA):** Arrays (duplicates, majority, kth largest), linked lists, trees/BST (traversal, invert/mirror, AVL), recursion/DP (LCS, climbing stairs), graphs (bipartite), hashing, string permutations/palindromes.
- **CS fundamentals:** C internals (`malloc` vs `calloc`, what `#include` does); Java exceptions / Java vs C++; DS comparisons across languages; operator nuance. Recent reviews also mention SQL/DBMS operators.
- **Projects:** Resume projects (including light LLD-style discussion) appear in later technical rounds.
- **Puzzles:** Recurring (chessboard squares, ball-probability, 100 doors, “average salary without revealing individuals”).
- **HR (when present):** Why Fiorano, intro, casual fit — often light.
- **HLD:** Not a dedicated stage; one Library Management design question appeared as an outlier. Domain ESB/JMS questions are **not** confirmed in fresher interviews (JD signals only).

|---|---|---|---|---|---|---|---|---|
| High | Low–Med | Med–High | Low–Med | Low | Low | None | Low | High |

## Recurring Patterns & Tips
- OA MCQ density is the first hard filter — GFG-style MCQ practice helps.
- Interview DSA spans lists/trees/DP more than Hard contest graphs.
- Puzzles show up often enough to prep a small set.
- Do not over-prep Fiorano product-domain (JMS/ESB) for campus SWE unless the JD says so.
