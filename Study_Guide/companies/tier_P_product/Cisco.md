## Process Overview
Cisco India early-career software hiring mixes **campus drives**, national **Code with Cisco**, and careers-portal / apprentice posts. Stage count varies by channel and Business Unit.

| Path | Entry | Typical flow |
|---|---|---|
| Campus FTE / Intern | TPO / pool campus | OA → inclusive (masked) resume → Webex interviews → ETR/HR |
| Code with Cisco | Register via ETR page / TPO | Common Entrance OA → optional onsite code-a-thon → ~4 interview rounds → LOI |
| Off-campus / apprentice | careers.cisco.com | Posting-dependent; CwC is the main structured nationwide path |

**Campus vs CwC vs off-campus:** Campus often OA → 2 technical → ETR/HR; CwC officially describes four virtual rounds; hackathon finalists still sit interviews (hackathon ≠ offer). Off-campus SWE posts are less uniform.

**M.E./M.Tech CS:** Fresher guides list B.E./B.Tech/M.Tech/MCA; Code with Cisco cycles may allow all branches when pass-out year matches — confirm each drive.

## Online Assessment (OA)
- **Platform:** HackerRank (widely reported for campus / CwC 2024–2025)
- **Duration:** ~90 minutes (official CwC FAQ)
- **Format:** ~40 MCQs + **2 DSA coding** problems (consensus)
- **Negative marking:** No (official CwC FAQ)
- **MCQ topics:** Strong **Computer Networks**; also OS, OOP, DBMS; aptitude; occasional ML/electronics
- **Coding themes:** trees/graphs, tries + BFS, traversal optimization — medium+
- **Proctoring:** Remote AI/video for CwC

## Interview Rounds
1. **Technical (~40–60 min)** — resume/projects; DSA; **heavy CN** (TCP/UDP, DNS, DHCP, HTTP/HTTPS, ARP…); OOP/OS/DBMS
2. **Technical 2 / Managerial** — deeper projects, more CN/OS, teamwork
3. **Executive (when present)** — senior grill; harder DSA discussion; security/networking scenarios
4. **ETR / HR (~10–15 min)** — locations, higher studies, why Cisco, culture fit

## Recurring Patterns & Tips
- Know every resume line; expect scale/security follow-ups.
- CN + OS clarity matters as much as DSA.
- Inclusive resume / candidate codes — do not volunteer masked fields.
- Code-a-thon is visibility, not a bypass of interviews.
- Prefer BU/role-specific answers over generic “networking company” talk.
