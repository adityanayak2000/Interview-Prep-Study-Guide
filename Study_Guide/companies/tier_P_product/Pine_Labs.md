## Process Overview
Pine Labs (fintech / merchant payments) hires across **Noida, Bengaluru, Pune**. Campus/intern loops are **OA-heavy with a short post-OA interview**; experienced off-campus loops are **DSA + LLD/HLD + backend fundamentals**.

| Path | Entry | Typical flow |
|---|---|---|
| Campus / intern / trainee | TPO / assessment email | Resume shortlist → OA (aptitude + coding) → short interview (~10 min) |
| Off-campus / referral (SDE-2) | Referral / Naukri / recruiter | Telephonic → 3–4 technical (DSA, LLD, director/manager) → HR |
| Senior C++ | Recruiter | Notepad coding + C++ internals + multithreading + HR |

**Campus vs off-campus:** Campus almost always has **quant + logical (+ English on some drives) + 3 medium coding**. Referral paths may **skip OA** and jump straight into hard DSA + RedBus-style LLD. Design depth scales with seniority.

**M.E./M.Tech CS:** No BITS-specific public write-up; IIT ME/M.Tech hire profiles exist. Expect same OA/interview bar as strong tier-1 campus SWE when called for full-time SDE.

## Online Assessment (OA)
- **Platform:** Unknown (assessment email; vendor not named in GFG reports)
- **Duration:** Unknown
- **Negative marking:** Unknown
- **Sections — SDE Trainee (GFG):** Quant (20) → Logical (20) → Coding (3 medium)
- **Sections — Intern variant (GFG):** Quant → Logical → English comprehension → Coding (3 medium: Stock Span, LL multiply-by-2, majority element)
- **Off-campus OA (weaker):** TheJobOverflow 2023 tagged LC-style sliding-window problem; full section map not documented
- **Difficulty:** Medium LeetCode-classic for coding; logical reasoning can be moderate–high

## Interview Rounds
### Campus / intern / trainee
1. **OA** — aptitude + coding
2. **Short interview (~10 min)** — intro, college project, logical puzzle (e.g. water jug)

### Experienced SDE-2 (referral / Naukri patterns)
1. **Recruiter telephonic**
2. **DSA** — can include **hard** problems (e.g. median of two sorted arrays); DP / binary search
3. **LLD** — booking systems (RedBus-like), SOLID, patterns
4. **Director / HM / HLD** — Spring/JVM/HashMap, multithreading, thread pools; or Spotify-style HLD
5. **HR** — culture / role fit

## Recurring Patterns & Tips
- Prep **two modes**: campus OA+light interview vs experienced DSA+design+Java/concurrency.
- OA coding themes: arrays, linked lists, stacks (Stock Span), majority element.
- Backend depth (transactions, JVM, thread pools) appears in senior rounds.
- Payments domain language (aggregators, POS throughput) helps in project/design talk.
- Notepad / no-IDE coding reported on some C++ tracks.
