## Process Overview
CRED hiring is selective and pipeline-driven rather than mass campus drives: targeted campus visits (e.g. newer IITs), intern cohorts with intern→FTE conversion, CodeChef contest + resume screening, referrals/LinkedIn, and recruiter outreach. Timeline often **1–3 weeks** once rounds start; campus days may compress multiple rounds into one day.

**M.E./M.Tech CS:** No degree-specific first-person reports found. Aggregators list M.Tech CS/IT as eligible; once shortlisted, PG candidates face the same bar (build quality + design over pure DSA speed). Realistic paths: intern cohort → FTE, off-campus contest/referral, or selective campus visit — not a typical mass placement-day slot.

| Track | Entry | Round shape |
|---|---|---|
| Campus backend (select visits) | On-campus day | HackerRank coding → HackerRank project → live technical → CRED Mark |
| Intern / new-grad | Contest and/or OA → project / machine coding | 4 virtual rounds or 2.5 hr machine coding after contest |
| Off-campus SDE / SDE-2 | Recruiter / referral | Machine coding → DSA → HLD → HM **or** take-home → DSA → MC → design |
| Android SDE (off-campus) | LinkedIn apply | 24 hr take-home → DSA → domain → design + HoE |

Campus vs off-campus: campus often uses **HackerRank OA (90 min, 3 problems)** then project round; off-campus may use **24 hr take-home** or skip OA (experienced).

## Online Assessment (OA)
**Format is track-dependent.**

### Campus / intern (new grads)
- **Platform:** HackerRank (coding + project); CodeChef for some intern off-campus cohorts
- **Duration:** Coding ~90 min; project ~90–150 min
- **Format:** 3 coding problems and/or single backend LLD project on skeleton
- **Topics:** Combinatorics, bitmask DP, greedy, ad-hoc, graphs/strings — often CP-flavored, not only LeetCode templates
- **Difficulty:** Medium–Hard; strong filter
- **Negative marking:** Unknown

### Off-campus experienced
Often **no OA**; may start with recruiter screen or 24 hr take-home.

## Interview Rounds
1. **Screen** — OA / contest / resume
2. **Machine coding / project (highest signal)** — 90 min–2.5 hr; parking lot, payments, booking, blogging workflow, LRU; SOLID, tests, extensibility, concurrency
3. **DSA / technical** — 60–90 min; medium problems or deep dive extending your project + DBMS/CAP/concurrency
4. **HLD (experienced / some SDE)** — storage, cache, payments, rewards; idempotency and fault tolerance
5. **CRED Mark / HM / HoE** — values, projects, motivation; may blend HLD + behavioral

## Recurring Patterns & Tips
- Machine coding is the differentiator — practice timed LLD with tests.
- Campus OA is CP-flavored (bitmask DP, combinatorics).
- Fintech vocabulary: idempotency, double-charge prevention, ledger consistency.
- Resume depth matters in CRED Mark rounds.
- PG strategy: intern cohorts, contest visibility, backend portfolio, referrals.
