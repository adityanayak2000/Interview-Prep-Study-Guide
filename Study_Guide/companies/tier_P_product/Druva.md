## Process Overview
Druva India engineering (SaaS data protection) is centered on **Pune**. Campus and fresher off-campus loops are **OA → technical → Techno-HR/managerial**. Experienced hires see longer design-heavy loops.

| Path | Entry | Typical flow |
|---|---|---|
| Campus FTE | TPO / on-campus visit | Presentation (sometimes) → OA → Tech → Techno-HR / Managerial |
| Off-campus fresher (2024 drive) | Careers / drive posting | Application → **Coderbyte** OA → technical interview(s) (Zoom) |
| Lateral / Staff SWE | LinkedIn / careers | Mettl-style OA → coding → design → HM / cultural |

**Campus vs off-campus:** Campus OAs often omit a named platform; 2024 fresher off-campus posts explicitly name **Coderbyte**. Round count post-OA is usually **2–3** for freshers vs **4–5** for experienced. Official careers: recruiter → hiring manager → panel; Zoom-first, possible Pune onsite later.

**M.E./M.Tech CS:** 2024 off-campus eligibility lists **BE/B.Tech/ME/M.Tech/MCA/M.Sc** (CS/IT-related). No separate PG track found — same OA + technical bar as UG. No BITS-specific public write-up; patterns from other campus visits.

## Online Assessment (OA)
Format **varies by channel** — do not invent a single template.

- **Platform (sourced; varies):** Campus — often unnamed; Off-campus fresher 2024 — **Coderbyte**; Experienced — **Mettl** reported
- **Duration:** ~70–100 min campus reports; Mettl experienced ~75 min; follow invite email
- **Sections & format (reported variants):**
  - Campus: **MCQs + 3 coding** (easy–medium); MCQ topics DSA, CN, OS, DBMS, C/C++
  - Off-campus 2024: MCQs + coding challenges on Coderbyte
  - Experienced Mettl: **3 coding + 15 MCQs** (C++, Networking & OS, DSA)
- **Negative marking:** Unknown (no source)
- **Topics tested:** strings/arrays/LL/trees/DP; OS (Linux, IPC); CN (OSI, ports, attacks); DBMS indexing
- **Difficulty:** Easy–medium for fresher coding; selection often needs strong coding + fundamentals

## Interview Rounds
1. **OA** — gate (format above)
2. **Technical 1** — resume walkthrough; **2–3 DSA** (LL, stack, arrays, trees, DP); HTTP/REST, SQL vs NoSQL, TCP/UDP, OSI; code quality scored
3. **Technical 2 / Techno-HR** — OS (deadlock, semaphores, IPC, scheduling); CN (ARP, latency); OOP; projects; sometimes recursion + leadership behavioral
4. **HR / Managerial (when present)** — why Druva, hobbies, team fit

**Experienced add-ons (not typical ME campus):** system design (rate limiter, sharding), concurrency (threaded pagination), cultural fit with director.

## Recurring Patterns & Tips
- Weak OS/CN with strong DSA is a common rejection pattern — prep both.
- Narrate approach; interviewers score **code quality** and complexity discussion.
- Resume is a question bank — defend every line.
- Concurrency (threads, mutex) can appear even in older fresher reports.
- Prefer official `@druva.com` channels; careers page warns about fraud.
